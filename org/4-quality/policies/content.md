# Content Policy

> **Applies to:** All externally-facing content — documentation, blog posts, marketing materials, press releases, website copy, social media, presentations  
> **Enforced by:** Quality Layer eval agents  
> **Authority:** Content & Communications leads

---

## Principles

1. **Accuracy over speed** — Never publish claims without evidence.
2. **Audience-appropriate** — Match tone, depth, and terminology to the target audience.
3. **Brand-consistent** — Follow brand voice guidelines.
4. **Grounded** — All technical claims must be reproducible and verifiable.

## Mandatory Requirements

### Accuracy & Claims
- [ ] All technical claims verified against current product capabilities
- [ ] Performance claims backed by reproducible benchmarks
- [ ] Competitive claims factual and documented
- [ ] No forward-looking statements without proper disclaimers
- [ ] Statistics cited with sources

### Brand & Voice
- [ ] Follows brand voice guidelines (tone, personality, values)
- [ ] Company and product names spelled correctly and consistently
- [ ] Approved terminology used (per glossary)
- [ ] No deprecated product names or branding
- [ ] Consistent capitalization and formatting

### Documentation Quality
- [ ] Follows documentation taxonomy (tutorial, how-to, concept, reference)
- [ ] Code examples tested and working
- [ ] Screenshots current (not outdated UI)
- [ ] Links validated (no broken links)
- [ ] Version-specific content clearly labeled

### Legal & Compliance
- [ ] No confidential information disclosed
- [ ] Trademarks properly attributed
- [ ] Open source licenses properly cited
- [ ] Privacy-compliant (no PII in examples)
- [ ] Approved by legal for external publication (if required)

### SEO & Discoverability (web content)
- [ ] Title and meta description optimized
- [ ] Heading hierarchy correct (H1 → H2 → H3)
- [ ] Alt text for all images
- [ ] Internal links to related content
- [ ] Canonical URLs configured

## Evaluation Criteria

| Criterion | PASS | FAIL |
|-----------|------|------|
| Technical accuracy | All claims verified | Unverified or false claims |
| Brand consistency | Follows guidelines | Off-brand voice or terminology |
| Documentation type | Clearly classified | Mixed types in one page |
| Code examples | Tested and working | Untested or broken code |
| Legal compliance | Reviewed and approved | Confidential info exposed |
