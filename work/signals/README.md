# Signals

Incoming observations from market, customer, technical, internal, competitive, and financial sources.

## How to Create a Signal

1. Copy the template: `process/templates/signal.md`
2. Name the file: `YYYY-MM-DD-<descriptive-name>.md`
3. Fill in the template
4. Submit as a Pull Request

## Signal Categories

| Category | Examples |
|----------|----------|
| **Market** | Industry trends, analyst reports, regulatory changes |
| **Customer** | Feature requests, support trends, churn signals, NPS |
| **Technical** | Production incidents, tech debt, performance degradation |
| **Internal** | Team feedback, process friction, division gaps |
| **Competitive** | Competitor launches, pricing changes, market moves |
| **Financial** | Revenue trends, cost anomalies, margin changes |

## Dispositions

- **Proceed** → Signal moves to opportunity validation in Discover loop
- **Defer** → Signal kept, revisited in next planning cycle
- **Monitor** → Signal tracked, follow-up date set
- **Archive** → Signal not actionable, kept for reference
