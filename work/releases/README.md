# Releases

Release contracts for shipped software. Each release is documented using the release contract template.

## How to Create a Release Contract

1. Use the template: `process/templates/release-contract.md`
2. Name the file: `YYYY-MM-DD-<release-name>.md`
3. Submit as a Pull Request for Ship Loop approval

## Naming Convention

- `YYYY-MM-DD-<descriptive-release-name>.md`

## Release Statuses

| Status | Meaning |
|--------|---------|
| **draft** | Release contract being prepared |
| **approved** | Approved for deployment |
| **deploying** | Progressive rollout in progress |
| **deployed** | Successfully deployed to production |
| **rolled-back** | Deployment rolled back |
