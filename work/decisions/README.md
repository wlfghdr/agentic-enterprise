# Decisions

Architecture, strategy, and process decisions. Each decision is documented using the decision record template.

## How to Create a Decision Record

1. Use the template: `process/templates/decision-record.md`
2. Name the file: `YYYY-MM-DD-<descriptive-name>.md`
3. Submit as a Pull Request for review
4. Architecture decisions require Architecture Governor review

## Decision Categories

| Category | Examples | Reviewer |
|----------|----------|----------|
| **Architecture** | Service boundaries, API patterns, data models | Architecture Governor |
| **Strategy** | Market positioning, investment priorities | Steering Layer |
| **Process** | Workflow changes, policy updates | Process Owner |
| **Technology** | Tool selection, framework adoption | Tech Leads |

## Decision Statuses

| Status | Meaning |
|--------|---------|
| **proposed** | Under review |
| **accepted** | Approved and in effect |
| **deprecated** | No longer recommended (superseded or obsolete) |
| **superseded** | Replaced by a newer decision (link to successor) |
