# PII Inventory Entry: [Data Category Name]

> **Template version:** 1.0 | **Last updated:** 2026-03-14
> **Tracks personal data processing activities.** Each entry describes one category of personal data, its classification, processing purpose, and handling controls. Required by the [Data Classification & Handling Policy](../../org/4-quality/policies/data-classification.md) and the [Privacy Policy](../../org/4-quality/policies/privacy.md).

---

## Data Category

| Field | Value |
|-------|-------|
| **Entry ID** | _(e.g., PII-2026-001)_ |
| **Data category** | _(e.g., customer email addresses, employee health records, billing information)_ |
| **Classification** | _(CONFIDENTIAL / RESTRICTED — PII is minimum CONFIDENTIAL per policy)_ |
| **Special-category data** | no _(yes if GDPR Art. 9: racial/ethnic origin, political opinions, religious beliefs, trade union membership, genetic data, biometric data, health data, sex life, sexual orientation)_ |

## Processing

| Field | Value |
|-------|-------|
| **Processing purpose** | _(e.g., account authentication, billing, customer support, analytics)_ |
| **Lawful basis** | _(consent / contract / legal obligation / vital interests / public task / legitimate interest — per privacy.md)_ |
| **Role** | _(controller / processor / joint controller)_ |
| **Data subjects** | _(e.g., customers, employees, website visitors, partners)_ |
| **Source** | _(e.g., collected directly, derived, received from third party)_ |
| **Automated decision-making** | no _(yes if used for profiling or automated decisions — triggers DPIA per privacy.md §5)_ |

## Storage & Retention

| Field | Value |
|-------|-------|
| **Storage location(s)** | _(e.g., primary database, backup, analytics warehouse — include regions/jurisdictions)_ |
| **Encryption at rest** | _(required per classification level — reference cryptography.md)_ |
| **Retention window** | _(e.g., duration of contract + 30 days, 7 years for financial records)_ |
| **Deletion method** | _(verified deletion / cryptographic erasure — per classification handling matrix)_ |
| **International transfers** | _(none / list destination countries + transfer mechanism per privacy.md §7)_ |

## Access & Sharing

| Field | Value |
|-------|-------|
| **Authorized teams / divisions** | _(which teams have access)_ |
| **Authorized agent types** | _(which agent types may process this data)_ |
| **Third-party sharing** | _(none / list recipients + DPA status per privacy.md §2)_ |
| **Access audit logging** | _(required for CONFIDENTIAL+)_ |

## Compliance Links

| Field | Value |
|-------|-------|
| **DPIA reference** | _(link to DPIA if applicable, or "not required" with justification)_ |
| **DPA reference** | _(link to DPA if processor relationship exists)_ |
| **Related asset registry entry** | _(link to asset entry if data is part of a registered asset)_ |
| **Related mission** | _(mission ID if this data was introduced as part of a mission)_ |

## Ownership

| Field | Value |
|-------|-------|
| **Data owner** | _(team or individual responsible for classification and handling decisions)_ |
| **Privacy contact** | _(privacy/legal contact for this data category)_ |
| **Last review date** | YYYY-MM-DD |
| **Next review date** | _(at minimum quarterly)_ |
